Postman ile Backend API Otomasyonunda Dinamik Değişkenler

Herkese merhaba,

Serinin dördüncü yazısında sizlerle Postman ile backend API otomasyonunda dinamik değişkenler hakkında bilgiler vereceğim. Burada dinamik değişken tiplerinden ve değişken kapsamlarından bahsederek küçük kod örnekleri göreceğiz. Keyifli bir yazı olması dileğiyle…

Hadi başlayalım!



Değişkenler, dinamik değerleri depolamak için kullanılan statik tanımlayıcılar olarak bilinir. Değişkenlerin değerlerini değiştirebilir ve birden fazla yerde kullanabilirsiniz. Değeri değiştirirseniz, değişkenin her örneği buna göre değişir, tekrarlayan değişikliklere olan ihtiyacı ve hata olasılığını azaltırsınız. Postman’de değişkenlerin tutulduğu ortamlar environment ve globals ortamlarıdır.

1. Environments

Environment, ortam değişkenleri oluşturabildiğimiz ve bu değişkenleri sakladığımız bir alandır. Environment altında bulunan değişkenleri, farklı request veya collection’larda çağırabiliriz. Bu değişkenleri pre-request ve test alanlarında set veya get edebiliriz.



Environment, bir key-value çiftleri kümesidir. Key, değişkenin adını temsil eder. Örneğin; API’lerle çalışırken, yerel makineniz, test ortamınız ve prod ortamınız farklı IP’lere sahiptir. Request’lerinizi değiştirmeden farklı ortamlar arasında geçiş yapmak için değişkenleri kullanarak request’leri özelleştirebilirsiniz.

Bir environment oluşturabilir, paylaşabilir, çoğaltabilir, dışa aktarabilir, silebilir ve tek bir JSON dosyası olarak içe aktarılabilir.


2. Globals

Global değişkenler, tüm kapsamlarda her zaman kullanılabilen bir değişkenler kümesi sağlar. Birden fazla environment’a sahip olabilirsiniz, ancak bir global değişken kümesine sahip olursunuz.



3. Değişkenlerin Kapsamı

Postman’de değişkenlerin kapsamları birbirinden farklıdır. Beş tür değişken kapsamı atayabilirsiniz:

  1. Global
  2. Collection
  3. Environment
  4. Data
  5. Local

Aynı isme sahip bir değişken iki farklı kapsamda ise, önceliği yüksek olan kapsam önceliklidir. Aşağıda bu beş değişken türü için kapsamlar gösterilmiştir.



Değişkenler, kod ile oluşturulabilir, set veya get edilebilir. Dinamik olarak url, headers, request body, pre-request ve test script alanlarında kullanabilir.

Örneğin; pre request scripts alanında set edilen bir değer request body alanında kullanılabilir. Bu işlemi aynı collection içerisinde bulunan farklı API veya folder’lar içinde yapabilirsiniz.


4. Değişkenler (Variables)

Tüm değişkenler Postman GUI’sini kullanarak manuel olarak ayarlanabilir ve scope dahilindedir. Değişkenler komut dosyalarındaki kod parçacıklarında kullanılabilir (pre-request, tests).

Request Alanındaki Değişkenlerin Get Edilmesi

Syntax: {{myVariable}}

Örnekler:

  1. Request URL: http://{{domain}}/users/{{userId}}

  2. Headers (key:value): X-{{myHeaderName}}:foo

  3. Request Body: {“id”: “{{userId}}”, “name”: “Ozan Eser”}

4.1. Global Değişkenler (Global Variables)

Global değişkenler diğer requestlere veri aktarmak amacıyla kullanılır. Gerekmediğinde değişkenleri silmenizi öneririm.

4.1.1. Değişkeni Set Etme

pm.globals.set(‘myVariable’, MY_VALUE);


4.1.2. Değişkeni Get Etme

pm.globals.get(‘myVariable’);

Alternatif olarak;

pm.variables.get(‘myVariable’);


4.1.3. Global Değişkeni Kaldırma

  1. Bir Global Değişkeni Kaldırma

    2. pm.globals.unset(‘myVariable’);

  2. Tüm Global Değişkenleri Kaldırma

    3. pm.globals.clear(‘myVariable’);



4.2. Collection Değişkenleri (Collection Variables)

Genellikle collection’un çalışması sırasında değişmeyen bazı sabitleri (URL ve authentication credentials) saklamak için kullanılır.


4.2.1. Collection Değişkeni Set Etme

Collection değişkenleri belirli bir collection’a bağlanır ve yeni değişkenler yalnızca Postman GUI’si kullanılarak tanımlanabilir veya değiştirilebilir.


4.2.2. Collection Değişkeni Get Etme

pm.variables.get(‘myVariable’);


4.2.3. Collection Değişkeni Kaldırma

Collection değişkenleri yalnızca Postman GUI’sinden kaldırılabilir.



4.3. Ortam Değişkenleri (Environment Variables)

Ortam değişkenleri seçilen ortama bağlıdır. Global değişkenlerden daha dar bir kapsama sahip olmalarına rağmen iyi bir alternatiftir. Diğer requestlere veri iletmek, ortama özel bilgileri depolamak ve collection’ların çalışması sırasında değişmeyen URL ve authentication credential’lar için kullanılır.


4.3.1. Ortam Değişkenlerini Set Etme

pm.environment.set(‘myVariable’, MY_VALUE);


4.3.2. Ortam Değişkenlerini Get Etme

pm.environment.get(‘myVariable’);

En yakın kapsama bağlı olarak;

pm.variables.get(‘myVariable’);


4.3.3. Ortam Değişkeni Kaldırma

  1. Bir Ortam Değişkeni Kaldırma

    2. pm.environment.unset(‘myVariable’);

  2. Tüm Ortam Değişkenleri Kaldırma

    3. pm.environment.clear();


Örnekler:

pm.environment.set(‘name’, ‘Ozan Eser’);

console.log(pm.environment.get(‘name’);

console.log(pm.variables.get(‘name’));


4.4. Veri Değişkenleri (Data Variables)

Yalnızca bir Collection Runner ve Newman tarafından yaratılan iteration’lar sırasında gerçekleşir. Birden fazla veri setinin gerektiği durumlarda kullanılır.


4.4.1. Veri Değişkenlerini Set Etme

Sadece bir CSV veya JSON dosyasından set edilebilir.


4.4.2. Veri Değişkenlerini Get Etme

pm.iterationData.get(‘myVariable’);

En yakın kapsama bağlı olarak;

pm.variables.get(‘myVariable’);


4.4.3. Veri Değişkenini Kaldırma

Yalnızca CSV veya JSON dosyasının içinden kaldırılabilir.


4.5. Yerel Değişkenler (Local Variables)

Yerel değişkenler, yalnızca yaratıldıkları requestte tanımlıdır. Pre request script’ten test script’e veri aktarılmasında kullanılır.


4.5.1. Yerel Değişkenlerini Set Etme

pm.variables.set(‘myVariable’, MY_VALUE);


4.5.2. Yerel Değişkenlerini Get Etme

pm.variables.get(‘myVariable’);


4.5.3. Yerel Değişkeni Kaldırma

Testler tamamlandıktan sonra yerel değişkenler otomatik olarak kaldırılır.


4.6. Dinamik Değişkenler (Dynamic Variables)

Yalnızca Request oluşturulurken kullanılabilir. Her request için yalnızca bir değer üretilir.

Example: JSON body:



Ayrıca Postman versiyon 7.2 ile faker.js adlı kütüphane kullanılmaya başlanıldı. Bu kütüphane sayesinde bir çok yeni dinamik değişken eklenebilmektedir. Bu değişkenler ile her request için farklı değerler oluşturulmaktadır. Bu dinamik değişkenleri incelenizi ısrarla tavsiye ederim.


4.7. Log ve Debug Değişkenleri (Logging and Debugging Variables)

Postman’den console log’u açın. Pre request veya test script kodunuza console.log kodunu yazın.

Örnekler:

var myVar = pm.globals.get(“myVar”);

console.log(myVar);

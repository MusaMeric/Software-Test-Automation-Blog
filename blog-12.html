JMeter Kurulum ve Farklı Çalıştırma Modları

Herkese merhaba,

Serinin ikinci yazısında sizlere JMeter kurulumu ve farklı çalıştırma modları hakkında bilgiler vereceğim. JMeter için sistem gereklilikleri nelerdir? Kullandığımız işletim sistemine göre kurulum adımları nasıl gerçekleştirilir? Kurulan dizin yapısındaki dosyaların özellikleri ve önemi nedir? JMeter nasıl başlatılır? GUI modu, Server modu ve Komut satırı modu nedir, nasıl kullanılır ve avantajları nelerdir gibi sorulara cevap arayacağız. Keyifli bir yazı olması dileğiyle…

Hadi başlayalım!



JMeter saf Java açık kaynak (open-source) test yazılımlardan biridir. Bu nedenle çalıştığımız işletim sistemi üzerinde JMeter’i çalıştırmak istiyorsak makinemizde JDK yüklü olmalıdır. Bu nedenle ilk gereksinim makinemize JDK yüklemek veya yüklü olduğunu kontrol etmek olacaktır. JMeter sistem gereksinimi için JDK 1.6 veya üstü yeterlidir.



Hadi adım adım JMeter’i yükleyip keşfetmeye başlayalım.

Adım 1: Makinemizde Java’nın yüklü olup olmadığını kontrol edelim. Bunun için komut satırında ve işletim sistemimize bağlı olarak aşağıdaki java komutlarından birini yürütelim.

  1. Windows: c:\> java –version

  2. Linux: $ java –version

  3. Mac: machine: ~ ozan$ java –version

Eğer sisteminizde Java yüklüyse, üzerinde çalıştığımız işletim sistemine göre uygun bir çıkış elde ederiz. Örneğin Windows için;



Eğer Java yüklü değilse aşağıdaki linkten Java Software Development Kit’i (SDK) indirmeliyiz.

https://www.oracle.com/java/technologies/javase-downloads.html

Kurulumu tamamladıktan sonra kurulumun başarılı olup olmadığını yine yukarıdaki java –version komutu ile kontrol edebiliriz.

Adım 2: JMeter’ın son sürümünü http://jmeter.apache.org/download_jmeter.cgi adresinden indirelim. Sıkıştırılmış dosyayı çıkartalım ve sistemimizde uygun bir yerde tutalım.

Not: Aşağıdaki şekilde gösterildiği gibi indirmek için Binaries dosyasından (zip veya tgz) seçmeliyiz.



Adım 3: JMeter’ı kuralım. JMeter’in kurulumu son derece kolay ve basittir. zip veya tgz dosyasını, JMeter’in yüklenmesini istediğimiz dizine açmamız yeterlidir. Bizi uğraştıran sıkıcı bir kurulum ekranı yok! Basitçe unzip ve bitti.

Unzip işlemi tamamlandıktan sonra kurulan dizin yapısı aşağıdaki şekilde görünmelidir.



Burada ana dizin, JMeter script klasörleri ve JMeter yardımcı kütüphanelerinin olduğu dosya yapıları gösterilmiştir.

JMeter dizini birçok dosya ve dizin içermektedir. Aşağıda JMeter dizin ve dosyaları açıklanmıştır.

  1. /bin : JMeter’i başlatmak için kullanılan JMeter komut dosyasını içerir.

  2. /docs : JMeter dökümantasyon dosyalarıdır.

  3. /extras : İlgili ek dosyaları içerir.

  4. /lib/: JMeter için gerekli Java kütüphanelerini içerir.

  5. /lib/ext : JMeter ve protokoller için temel .jar dosyalarını içerir.

  6. /lib/junit : JMeter için kullanılan Junit kütüphanesidir.

Adım 4: JMeter’ı başlatalım. JMeter’i 3 modda başlatabiliriz. Bunlar;

  1. GUI Modu

  2. Server (Sunucu) Modu

  3. Command Line (komut satırı) Modu


JMeter’i GUI Modu ile Başlatma

JMeter’ı GUI modunda çalıştırmak için işletim sistemimize bağlı olarak bin dizini altından aşağıdaki dosyalardan bir tanesini çalıştırmalıyız.

  1. Windows : jmeter.bat

  2. Linux : jmeter.sh

  3. Mac : jmeter.sh

Yazının bundan sonraki aşamalarında Windows işletim sistemini kullanarak devam edeceğiz. Windows için jmeter.bat dosyasını çalıştırdığımızda bir Swing uygulaması olan JMeter GUI’si ile karşılaşırız. Bu GUI iki ana bileşenden oluşur. Bu bileşenler; ekranın sol tarafında test planlarını oluşturduğumuz bölme ve ekranın sağ tarafında test planlarını yapılandırdığımız bölmelerdir.



JMeter’i GUI Olmayan Mod ile Başlatma

1- JMeter’i Server Modu ile Başlatma

Server modu, dağıtılmış (distributed) testler için kullanılır . Bu test, client-server modeli olarak çalışır.

JMeter, client makinemizin performans açısından sunucumuzu streslendirecek kadar kullanıcıyı simüle edemeyebilir veya network düzeyimiz buna izin vermeyebilir. Tek bir JMeter istemcisi ile birden fazla uzak JMeter makinesini kontrol edebiliriz. JMeter’i uzaktan çalıştırarak, birçok alt uç bilgisayar arasında bir testi çoğaltabilir ve böylece sunucuda daha büyük bir yük simüle edebiliriz. Bu modelde, JMeter bir sunucu bilgisayarında sunucu modunda, client bilgisayarında ise JMeter GUI modunda çalışır.



JMeter’i sunucu modunu başlatmak için, bat dosyasını aşağıdaki şekilde çalıştırmalıyız (jmeter-server.bat).



2- JMeter’i Komut Satırı Modu ile Başlatma

JMeter GUI modunda kaydedilen komut dosyalarını çalıştırırken memory ve CPU tüketerek test sonuçlarımızı etkiler. Özellikle JMeter’da birden fazla listener çalıştırıyorsak, JMeter performansı önemli derecede etkilenir. Böyle bir durumun üstesinden gelmek için komut dosyasının GUI olmayan modda çalıştırılması gerekir. GUI olmayan modda JMeter’dan saniyede daha fazla istek alınır. Bu sebeple GUI olmadan JMeter’ı çalıştırmak tercih edilen bir seçenektir. Bunu aşağıdaki komut seçeneklerini kullanarak gerçekleştirebiliriz.

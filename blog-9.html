Postman ile Backend API Otomasyonunda İş Akışları (Workflows)

Herkese merhaba,

Serinin beşinci yazısında sizlerle Postman ile backend API otomasyonunda collection runner, iş akışları, rastgele veri oluşturma, requestler arasında diziler ve nesnelerin nasıl iletildiği gibi önemli konular hakkında bilgiler vereceğim. Keyifli bir yazı olması dileğiyle…

Hadi başlayalım o zaman!



1. Collection Runner

En kısa haliyle, collection runner test senaryolarımızı otomatize ederek yürütmemizi ve sonuçları ayrıntılı şekilde gözlemlemizi sağlamaktadır.

Collection, test senaryolarımızı gerçeklediğimiz bir dizi request grubundan oluşur. Bir collection 1 tane request’ten oluşabileceği gibi 1000 tane request’ten de oluşabilir. API testini otomatize etmek istiyorsanız bir collection’ı yürütmeniz gerekmektedir. Collection’ı yürüttüğünüzde altında bulunan bütün request’ler sırayla çalışmaktadır. (Eğer belirli durumlarda atlamak istediğiniz requestler varsa veya bu test senaryosunu sonlandırmak istediğiniz durumlar olursa postman.setNextRequest("request_name"); işlevi ile bunu kolayca gerçekleştirebilirsiniz.

Pre request ve test script’lerini kullandığınızda test suite’leri oluşturabilir, API request’leri arasında veri iletebilir. Gerçek API kullanım durumlarını yansıtan iş akışları oluşturabilirsiniz.

Collection’ı çalıştırmak için;

  1. Header bar’da bulunan “Runner” butonuna basın.
  2. Yürütmek istediğiniz collection’ı seçin.
  3. Environment seçin. (Diğer seçenekleri kendi kullanımınıza göre değiştirebilirsiniz.)
  4. Run Collection butonuna basarak test senaryonuzu yürütün.


Not: Postman’deki test senaryolarınızı Newman yardımıyla komut satırından da yürütebilir hatta Jenkins’e entegre edebilirsiniz. Bu konuya bir başka yazıda değineceğim.

Bu süreçte REST API’leri ile örnekler yapmak isterseniz https://reqres.in/ sitesini takip edebilirsiniz. Bu şekilde request’ler atıp response’ları Postman üzerinden test edebilirsiniz.


2. İş Akışları (Workflows)

Postman’de bildiğiniz gibi collection’ları otomatikleştirmek için Collection Runner ve Newman’i kullanıyoruz.

Aşağıda iş akışını değiştiren kod parçaları sadece bir sonraki request’e etki eder. Bu sebeple Postman uygulaması içerisinden çalıştırılan bir request’e etki etmez. Hatta pre request script’e yazılsa bile mevcut request’i atlamaz.

Bu kod parçalarının en çok kullanıldığı durum; bir koşul gerçekleştiğinde veya gerçekleşmediğinde collection içerisindeki sıradaki request’e değil, başka bir request’e atlamak için kullanılır.

Yürütmek istediğiniz request hangisi olacak ise request’in ismi yazılarak o request’e atlanır.



Request’i ve Collection Run durdurulmak istenirse aşağıdaki kod parçası çalışırılır.

4. Request’ler Arasında Diziler ve Nesneler Nasıl İletilir?

Response’un JSON formatında olduğunu varsayarsak, aşağıdaki şekilde response’u bir değişkene atayabiliriz.



Bundan sonra tüm response’u set edebilirsiniz. Örneğin aşağıdaki şekilde bir ortam değişkenine atanabilir.



Bir sonraki istekte verileri aşağıdaki formatta alabiliriz.

Request body’de kullanılacaksa; {{myData}}

var myData = JSON.parse(pm.environment.get(‘myData’));


Not: Nesneleri ve dizileri bir değişkene kaydetmeden önce JSON.stringfy() komutunu kullanmamız gerekir. Aksi takdirde Postman veya Newman sürümüne bağlı olarak çalışmayabilir.

Değerler string, integer veya boolean ise JSON.stringify ve JSON.parse metodlarına gerek yoktur.

JSON.stringify(), herhangi bir Javascript nesnesini alır ve onu string formatına çevirir.

JSON.parse(), string formatındaki veriyi javascript object’e çevirir.


5. Access Token Nasıl Alınır? Başka Bir Request’te Nasıl Kullanılır?

Bir request’ten aşağıdaki gibi bir Response döndüğünü ve bizimde burda bulunan access Token’ı alıp başka bir yerde kullanmamız gerektiğini düşünelim. (Bu durum Postman ile API otomasyonu yazarken çok sık rastlayacağımız bir durumdur.)



Test sekmesine aşağıdaki kod parçasını yazarak 1. Satırda response’a erişir. İkinci satırda ise accessToken’ı, token isimli değişkene atarız.



Ardından aşağıdaki gibi bu değişkeni global veya ortam(environment) değişkenine atayarak başka request’ler için kullanabiliriz.

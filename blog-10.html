Postman ile Backend API Otomasyonunda Newman Entegrasyonu ve Html Reporter

Herkese merhaba,

Serinin altıncı yazısında sizlerle Postman ile backend API otomasyonunda Newman entegrasyonu ve Html Reporter hakkında bilgiler vereceğim. Keyifli bir yazı olması dileğiyle…

Hadi başlayalım o zaman!



Newman, Postman collection’larımızı Collection Runner üzerinden değil doğrudan komut satırı üzerinden koşmamızı ve test etmemizi sağlayan bir Npm (Node Package Manager) teknolojisidir. Npm javascript betik dili için geliştirilmiş olan ve Node.js’in standart olarak kabul ettiği bir paket yönetim sistemidir. Npm komut satırından çalıştırılır. Bu sayede CI (Continuous Integration) sistem ve server’lara kolayca kurabiliriz (Jenkins, Travis CI, Gitlab,Bamboo). Burada Newman diğer tool’lara ulaşmak için kullanılan bir gateway gibi hayal edilebilir.



Aşağıdaki adımları takip ederek Windows işletim sistemi için Newman entegrasyonunu tamamlayabilirsiniz.

  1. Postman Collection’ları Newman ile komut satırından çalıştırabilmemiz için öncelikle bilgisayarımıza node.js indirmemiz ve kurmamız gerekmektedir. https://nodejs.org/en/ sitesi üzerinden node.js indirin ve kurunuz.

  2. Node.js yükledikten sonra komut satırını (cmd) açınız. Newman’ı aşağıda yer alan komut ile kurunuz.

  3. Artık komut satırı üzerinden testlerinizi koşmaya hazırsınız. Koşacağımız collection’ı export ederek komut satırından o dosyanın bulunduğu dizine gitmeliyiz. Ardından aşağıdaki konumu çalıştırarak collection’ı test edebiliriz.

Ek olarak Newman’de hangi özellikleri kullanabileceğimizi görmek için newman run –h komutunu çalıştırabiliriz.



Benim burdaki özellikler arasında en çok kullandıklarım –n, -e ve –g özellikleridir. Bu özellikler;

n : collection’ı kaç kere koşacağımızı belirlediğimiz özelliktir. Örneğin 10 kere koşmak için;



e : Collection Runner’da olduğu gibi collection’da kullanılan ve environment dosyasında tanımlanan değişkenleri kullanmamıza olanak sağlar. Örn;



g : Collection’da kullanılan ve global dosyasında tanımlanan değişkenleri kullanmamıza olanak sağlar.Örn;






HTML Reporter

Newman ile komut satırından koştuğumuz testleri HTML formatında rapor haline getirmemize de olanak sağlamaktadır. HTML reporter özellikle Jenkins veya başka herhangi bir CI server kullanılırken çok elverişlidir. Fail olan testleri debug etmemizde bize yardım eder ve bize genel bir bakışla test sonuçlarını inceleme imkanı sunmaktadır.

Komut satırından aşağıdaki komutu çalıştırarak HTML reporter kurulumlarını tamamlayabilirsiniz.



Bir collection’ı koştururken aşağıdaki komutta olduğu gibi --reporters=cli,htmlextra özelliğini kullanırsanız belirlenen dizine html dosyası otomatik olarak oluşturulacaktır.
